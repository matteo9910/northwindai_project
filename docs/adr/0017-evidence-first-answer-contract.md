# Evidence-first answer contract

The agent's final synthesis is strictly evidence-first: every claim must trace to a specific evidence element recorded in `answer_trace` (a graph row, SQL row, or retrieved chunk), the answer explicitly separates certain facts from plausible relationships (ADR 0012), and no outside model knowledge is introduced. When evidence is insufficient the agent **abstains** — it states what it could not determine rather than filling the gap. For genuinely ambiguous or under-specified requests it returns a single targeted **clarification** question (a `needs_clarification` terminal state); for minor under-specification it makes a reasonable assumption and states it; out-of-domain questions are refused with an explanation.

**Consequences**: Abstention and clarification are first-class successful outcomes, not failures — the differentiator of a governed assistant over a chatbot. The synthesis runs on Sonnet 4.6 (ADR 0016) because the hard reasoning already happened during planning; its job is to write faithfully from the gathered evidence. Multi-turn conversation memory is out of scope for the first milestone: each request is independent, and a user's reply to a clarification arrives as a new request.
