# Agent evaluation strategy

The generic agent is evaluated with a curated suite of heterogeneous in-domain questions (beyond the ladder and the Golden Query), each carrying a behavioral expected-answer spec: expected route family, expected sources/tables used, facts that must appear (verified against a PostgreSQL/Neo4j read-back), required-abstention cases, and false-positive traps (drawn from the Controlled Scenarios, ADR 0011). Structural assertions run deterministically against the persisted `answer_trace` (route, valid generated queries, provenance present, evidence cited); an LLM-as-judge grades only the free-text synthesis against a rubric (grounded? abstains when it should? distinguishes plausible from certain?).

**Consequences**: This generalizes the ladder's behavioral-spec-plus-persisted-trace approach (Issue 11) to an arbitrary-question set, so it proves the agent is agnostic rather than tuned to one query. Evaluation is behavioral, never fixed-string — appropriate because the models reject `temperature` and outputs are non-deterministic (ADR 0016). The LLM-judge is confined to the prose; everything checkable deterministically stays deterministic. The Golden Query becomes one case in this suite, not the system's ceiling.
