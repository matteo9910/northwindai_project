# Enforce query guardrails in code

SQL and Cypher generated by AI Agent Query must pass through a dedicated validation layer before execution. Prompt instructions can guide the model toward read-only, scoped queries, but enforcement lives in code, such as a `query_validator.py` module between the agent and database executors.

**Consequences**: The agent may generate SQL or Cypher autonomously, but execution is allowed only after code-level checks for read-only operations, allowed schemas, allowed tables or labels, blocked mutation keywords, limits, and traversal constraints. This protects the system from prompt injection and keeps governance independent from model behavior.
